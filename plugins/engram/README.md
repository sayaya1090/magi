# engram — 자기개선 관찰자 플러그인

대화를 관찰해 **결과가 확정된 턴**에서 교훈과 검증된 스킬을 자동 추출·누적하고,
다음 세션에서 회상시켜 같은 시행착오의 반복을 막는다.
**engram**(엔그램)은 학습된 경험이 남기는 기억 흔적을 뜻한다. ds-cortex(OpenCode 자기개선 플러그인) 코어 로직의 Lua 포트로, 2026-07-13 magi에 탑재됐다.

## 무엇을 하나

- **교훈(lesson)**: 턴의 성패가 확정되는 순간, "어떤 작업을 / 어떻게 했더니 / 결과 / 교훈"을
  워크스페이스의 `SESSION_SUMMARY.md`(마크다운 표 원장)에 자동 기록한다.
- **스킬(skill)**: **검증된 성공**의 재사용 기법만 `.claude/skills/<slug>/SKILL.md`
  (skill-creator 규격 — OpenCode·Claude Code와 호환)에 자동 저장한다.
  에이전트가 제안만 한 것은 스킬이 되지 않는다.
- **회상(recall)**: 컨텍스트 프로바이더로 최근 교훈(≤50행/≤6000자)을 매 스텝 주입한다.
  두 파일을 git에 커밋하면 팀원이 받아 같은 실수를 피한다.

## 구조적 결과 게이트 (OpenCode 버전과의 핵심 차이)

OpenCode 어댑터는 사용자 발화 키워드("됐다/실패")로 성패를 **추측**하지만, magi는 턴의 성패를
**실측으로 안다**. `turn_finished` 관찰 이벤트의 `outcome` 필드가 게이트다:

| outcome | 의미 | 동작 |
| :--- | :--- | :--- |
| `verified` | 카운슬(3인 합의 게이트)이 실행 증거를 보고 done 합의 | 성공 확정으로 분석 → 교훈 + 스킬 후보 |
| `unverified` | 착지했지만 카운슬 미승인(딜록/비용캡/라운드캡) | 실패/부분 후보로 분석 |
| `guard` | loop/stall 가드 강제정지 | 실패 확정으로 분석 |
| `error` | 에러 종료 | 실패 확정으로 분석 |
| `done` | 판정 없는 평이한 종료(잡담 등) | 분석 안 함 |

판정은 `[호스트 판정]` 힌트로 사이드카 입력에도 주입돼 lesson.outcome이 실측과 모순되지 않게
앵커링된다. "검증된 성공만 스킬로"의 근거가 사용자 발화 추측 → 카운슬 합의로 격상된 것.

## 동작 방식

1. `magi.on("user_message"/"turn_finished")` 관찰 이벤트로 세션별 최근 대화 윈도우(≤8메시지)를 유지.
   이벤트는 **턴 경로 밖의 단일 워커**에서 돌므로 분석이 대화를 지연시키지 않는다.
2. outcome이 확정 계열이면 `magi.analyze`(도구 off 원샷 사이드카)로 교훈/스킬 JSON을 받는다.
   실패는 전부 무시(fail-soft) — 세션은 절대 중단되지 않는다.
3. 교훈은 원장에(세션 내·디스크 중복 가드), 스킬은 skill-creator 규격으로 저장.
   병합 판단용 기존-스킬 다이제스트는 플러그인 스토어(`skill_index`)에 유지한다
   (샌드박스에 디렉토리 나열이 없기 때문 — 스킬 디렉토리를 손으로 지우면 무해한 잔재가 남는다).
4. 사이드카가 도는 동안 이 플러그인의 컨텍스트 프로바이더는 그 스텝을 스킵할 수 있다
   (호스트의 바운드 락 획득이 의도한 우아한 퇴화).

## 설치 / 설정

**바이너리에 내장돼 있다** — 설치 없이 config 한 줄로 켠다(옵트인):

```toml
[plugins.engram]
enabled = true
```

업데이트는 `magi --update`(바이너리)에 실려 온다. 커스터마이즈하려면 이 디렉토리를 일반 플러그인 경로로 복사/링크하면 그쪽이 우선한다:

```bash
ln -s "<magi-repo>/plugins/engram" ~/.config/magi/plugins/engram   # 또는 <워크스페이스>/.magi/plugins/
```

사이드카 모델 오버라이드(선택, 생략 시 세션 모델):

```toml
# config.toml
[plugins.engram]
sidecar_model = "qwen3-coder:30b"
```

로드 확인: 디버그 로그에 `engram: 플러그인 로드 완료`.

## 필요한 호스트 프리미티브 (2026-07-13 이후 빌드)

- 관찰 이벤트 `user_message` / `turn_finished{session,text,outcome,reason}`
- `magi.analyze` (capability `"analyze"` — 매니페스트 선언 필수)
- `magi.json_decode`

## 품질 장치

- **중복 방지 3중**: 세션 내 키 가드 → 사이드카에 [기존 교훈] 다이제스트를 줘 재기록 거부 지시 →
  **결정론적 토큰 유사도 백스톱**(기존 교훈과 과반 겹치면 스킵; LLM 규칙은 소프트해서 실측상 뚫린다).
- **스킬 필드 확장**: `verify`(적용 후 확인 절차 — 200자 초과 시 `scripts/verify.sh`로 분리),
  `avoid`(쓰면 안 되는 경우/알려진 함정 — ❌실패 교훈 병합처). technique은 검증된 커맨드 원문 그대로.
- **사용 실적 계측**: 호스트가 turn_finished에 실어주는 "이번 턴 로드된 스킬" × 구조적 outcome으로
  스킬별 로드/성공/실패를 스토어(`skill_usage`)에 누적 — 병합·정리 판단의 실측 근거이며,
  [기존 스킬] 다이제스트에도 노출된다.
- **D13 합류**: 기록된 교훈/스킬을 `magi.propose_experience`로 공유 경험 스토어 리뷰 큐에도 제안
  (capability `experience`; 스토어 없으면 무시).

## 산출물 / 한계

| 경로 | 내용 | git |
| :--- | :--- | :--- |
| `SESSION_SUMMARY.md` | 교훈 원장 (OpenCode ds-cortex와 같은 포맷 — 파일 공유 호환) | 공유 대상 |
| `.claude/skills/<slug>/SKILL.md` | 자동 저장된 스킬 | 공유 대상 |

- **저장 알림 + 취소(N)**: 스킬 자동 저장 시 `magi.notify`로 세션에 알림이 뜨고, 다음 사용자
  메시지가 짧은 부정(`N`/`no`/`아니요`/`취소`/`스킬 취소` 등)이면 되돌린다(원샷 창 — 다른
  메시지가 오면 창이 닫힌다).
- 원장의 사용자 컬럼: `[plugins.engram] username` 설정 → 세션 사용자 라벨(`magi.set_user_label`,
  SSO 플러그인 등) → `"user"` 순으로 채운다.
- 스킬 후보 산출은 사이드카 모델 판단에 달린 best-effort; 산출 이후의 저장·중복 가드는 결정론적.
