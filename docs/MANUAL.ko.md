# magi 사용자 매뉴얼

[English](MANUAL.md) · [한국어](MANUAL.ko.md)

확장형 터미널 AI 코딩 에이전트 클라이언트. 공급자 무관(OpenAI 호환), 멀티에이전트,
Lua 플러그인, MCP, 공유 메모리를 지원한다.

---

## 1. 설치 & 요구사항

- **LLM 백엔드**: OpenAI 호환 엔드포인트([Ollama] 권장). 기본 모델은 **`gpt-oss:120b-cloud`** —
  Ollama **무료 클라우드 티어**라 GPU 없이 한 번만 로그인하면 된다.
  ```sh
  ollama signin                 # 무료 티어; 기본 gpt-oss:120b-cloud는 클라우드에서 실행
  # 완전 로컬로 돌리려면:
  ollama pull qwen3-coder:30b   # 가장 강한 로컬 코더 → ./magi --model qwen3-coder:30b
  ```
- **빌드**: `make build` 또는 `CGO_ENABLED=0 go build -o magi ./cmd/magi` (순수 Go 단일 바이너리)
- **프리빌트**: `curl -fsSL .../scripts/install.sh | bash` 또는 `brew install sayaya1090/tap/magi`

## 2. 실행

### 대화형 TUI
```sh
./magi                 # 다크/라이트 자동 감지
./magi --theme light   # 테마 강제(auto|dark|light)
```

### 헤드리스 (스크립트/CI)
```sh
./magi -p "list the go files and summarize"
./magi -p "create hello.txt with: hi" --output json   # JSONL 이벤트
echo "explain main.go" | ./magi -p -                   # stdin
```

헤드리스 출력 계약 (안정 — 스크립트·CI·벤치 어댑터가 의존):

- **종료코드**: `0` = 턴 정상 종료 · `1` = 에이전트 수준 에러로 종료(`loop_guard`,
  `stall_guard`, `max_steps`, 프로바이더 실패) · `2` = magi 자체가 프롬프트를 실행하지
  못함(셋업/제출 실패).
- **stdout** = 트랜스크립트: 모델 텍스트(최종 답), 툴 호출/결과, council·컴팩션 노트.
  `--output json`이면 fact 이벤트를 줄 단위 JSONL로 출력.
- **stderr** = 에러 전용. 에이전트 수준 에러는 grep 가능한 `error[<code>]: <message>` 형식.

**헤드리스 권한 거부는 가짜 사용자 결정이 아니라 정직하다.** 헤드리스에서 `--permission auto`/`ask`는 프롬프트에 답할 사람이 없으므로 `bash`/`webfetch`가 사용 불가다. 에이전트가 받는 툴 결과는 이를 **범주적으로** 말한다 — "이번 런에선 사용 불가(이 모드는 프롬프트 없이 승인 못 함), 재시도 말 것; 그 없이 진행하거나 왜 못 했는지 보고하라" — 인터랙티브 거부가 보낼 오해 소지 있는 `denied by user` 대신. 이 구분이 중요하다: `denied by user`는 "사람이 *이 호출*에 아니라고 했다"로 읽혀 에이전트가 변형을 재시도하며 헛돈다; 범주적 메시지는 그 능력이 런 전체에서 그냥 꺼졌다고 알려 한 번에 적응하게 한다. 그 툴이 필요하면 `--permission allow`(또는 인터랙티브 실행)를 쓰라. 기동 시 stderr에 이 불일치를 알리는 한 줄도 남는다.

### 환경 점검
```sh
./magi --doctor
```
`--doctor`는 magi가 필요로 하는 모든 것을 한 번에 진단하고 종료한다 — 새 머신이 이상하면 먼저 돌려라. 각 항목에 `ok`/`warn`/`fail` 한 줄을 출력한다:

- **LLM 엔드포인트** — `--base-url` 도달성과 설정된 `--model`이 거기 존재하는지(Ollama의 경우 클라우드 모델 로그인 여부).
- **선택 외부 도구** — `gopls`, `ast-grep`, `rg`(ripgrep): 있으면 검색/리팩터가 날카로워지고, 없어도 우아하게 degrade(warn일 뿐 fail 아님).
- **샌드박스 백엔드** — 사용 가능한 명령 격리(macOS `sandbox-exec`, Linux `bwrap` 등)와 bash가 격리 실행되는지.
- **Config** — **council 멤버 `provider`**가 정의된 `[llm.profiles.*]`를 가리키는지; 미정의면 warn(기본 백엔드로 폴백).

종료코드는 **하드 실패**(예: LLM 엔드포인트 도달 불가) 때만 0이 아니며, 경고는 advisory로 종료코드에 영향 없음.

### 버전 / 자동 업데이트
```sh
./magi --version
./magi --update            # 바이너리 + 관리형 플러그인 모두 업데이트 후 종료
./magi --update-core       # 바이너리만 업데이트(체크섬 검증)
./magi --update-plugins    # 관리형(git) 플러그인만 업데이트
./magi --plugin-install <git-url> [--plugin-pin <ref>]   # 플러그인을 사용자 플러그인 디렉터리로 clone
```

**관리형 플러그인** = git 체크아웃인 플러그인(`--plugin-install`로 설치했거나 직접 clone한 것).
`--update-plugins`는 각 플러그인에 fast-forward pull을 실행하며, 로컬 커밋/변경이 있어 ff 불가하거나
git remote가 없는 플러그인은 보고 후 **건너뜀 — 강제로 덮지 않음**. 직접 배치한 비-git 플러그인은
손대지 않음. 플러그인은 핫리로드되므로 `--update-plugins` 후 재시작 불필요.

**인터랙티브 기동 시 체크.** 터미널에서 TUI로 띄우면 최대 24시간에 한 번 새 릴리스를 확인함.
**패치** 릴리스는 한 줄 배너만 띄우고(`… magi --update 실행`), **마이너/메이저** 릴리스는 필수로 간주해
짧은 취소 가능 대기 후 자동 설치하고 재시작을 안내함. 이 체크는 **인터랙티브 TTY에서만** 발화 —
`-p`·파이프·CI·벤치마크에선 절대 안 됨 — 이라 비인터랙티브 실행은 네트워크 호출도, 갑작스러운 설치도
없음. `--no-update-check`(또는 `MAGI_NO_UPDATE_CHECK=1`)로 끔.

## 3. 설정

플래그/환경변수 (우선순위: 플래그 > 환경변수 > 기본값):

| 플래그 | 환경변수 | 기본값 | 설명 |
|---|---|---|---|

| `--base-url` | `MAGI_BASE_URL` | `http://localhost:11434/v1` | OpenAI 호환 base URL |
| `--permission` | `MAGI_PERMISSION` | TUI=`ask`/헤드리스=`allow` | `ask`\|`auto`\|`allow`\|`deny` |
| `--profile` | `MAGI_PROFILE` | (없음) | 가드레일 프리셋 `safe`\|`standard`\|`yolo` — 권한+샌드박스를 한 번에 설정(아래) |
| `--theme` | `MAGI_THEME` | `auto` | `auto`\|`dark`\|`light` |
| `--plugins` | `MAGI_PLUGINS` | (없음) | 추가 플러그인 디렉터리 |
| `--no-harness` | — | (꺼짐=하네스 켜짐) | 내장 하네스(포맷/진단/Stop 훅) 비활성화 |
| `--output` | — | `text` | `text`\|`json` (헤드리스) |
| `--time-budget` | `MAGI_TIME_BUDGET` | `0` (꺼짐) | 에이전트에게 가이드로 보여주는 소프트 벽시계 예산(예: `20m`). **권고일 뿐 하드 중단 아님**. §시간 & 스텝 예산 참고. 리더보드/비교 런에선 **끔** |
| `--workflow` | `MAGI_WORKFLOW` | (꺼짐) | 결정론적 localize→implement→verify→review 파이프라인으로 구동 |
| `--verify-cmd` | `MAGI_VERIFY_CMD` | (자동) | 워크플로 검증 명령; 비우면 자동 감지(go/cargo/npm/pytest 마커) |
| `--http-timeout` | `MAGI_HTTP_TIMEOUT` | `0` (무제한) | LLM 응답 헤더 최대 대기(예: `120s`) |
| `--no-cache` | `MAGI_NO_CACHE` | (꺼짐) | 프롬프트 `cache_control` 비활성화(기본 켜짐; 백엔드가 거부하면 자동 폴백) |
| `--list-models` | — | — | 백엔드의 사용 가능 모델 나열 후 종료 |
| `--doctor` | — | — | 환경 진단 후 종료(§환경 점검) |
| `--version` | — | — | 버전 출력 후 종료 |
| `--update` / `--update-core` / `--update-plugins` | — | — | 바이너리+플러그인 / 바이너리만 / 관리형 플러그인만 업데이트 후 종료 |
| `--plugin-install` / `--plugin-pin` | — | — | clone할 플러그인 git URL / 그에 대한 태그·브랜치·커밋(선택) |
| `--no-update-check` | `MAGI_NO_UPDATE_CHECK` | (꺼짐) | 인터랙티브 기동 시 업데이트 체크 비활성화 |

| — | `MAGI_REASONING_EFFORT` | (백엔드 기본) | reasoning 모델용 `reasoning_effort`로 백엔드에 전달 — 예: `none`으로 thinking 비활성, 또는 `low`\|`medium`\|`high`; 비우면 필드 생략 |
| — | `MAGI_EMOJI_WIDTH` | (자동 프로브) | 이모지 셀 폭을 강제: `narrow`\|`1`(1칸) 또는 `wide`\|`2`(2칸). 미지정 시 기동 프로브가 실측 |
| — | `MAGI_WIDTH_PROBE` | (켜짐) | `0`이면 기동 시 터미널 폭 프로브(ambiguous·데코·이모지)를 건너뜀 = 무보정(라이브러리 기본폭 사용) |
| — | `MAGI_AMBIGUOUS_WIDTH` | `auto` | `wide`\|`narrow`\|`auto` — 동아시아 ambiguous 문자 셀 폭 강제(아래 참고) |

권한 모드: `ask`=매번 확인 · `auto`=**편집은 자동 승인, 명령(bash)/네트워크만 확인** · `allow`=전부 자동 · `deny`=차단. TUI에서 `Shift+Tab`(또는 `/permission`)으로 순환.

가드레일 posture(`--profile`/`MAGI_PROFILE`)는 두 축(**권한 승인** × **OS 샌드박스**)을 한 번에 잡는 프리셋이다: `safe`=`ask`+`read-only`, `standard`(권장)=`auto`+`workspace-write`(편집 자동승인·명령/네트워크 확인·쓰기는 워크스페이스로 제한), `yolo`=`allow`+`full`. `--permission`/`sandbox`를 따로 주면 프리셋보다 우선한다. profile 미지정 시 샌드박스는 opt-in(무제한)으로 남고 권한만 기본 적용된다 — 기존 사용자의 네트워크/트리 밖 쓰기를 조용히 끊지 않기 위함. 샌드박스 축(`sandbox = "read-only"|"workspace-write"|"full"`)은 `config.toml`에서 직접 지정할 수도 있다.

**세밀 규칙(`config.toml`).** 모드 외에 세 리스트 키로 정책을 좁힌다: `allow`/`deny`는 툴 호출에 대한 글롭 규칙(예: `Bash(git push:*)`는 그 명령 자동승인, `Read(**/.env)`는 시크릿 읽기 차단)으로 — `p` 권한 선택(§4)이 영속하는 것이 바로 이것 — `deny`가 `allow`보다 우선한다. `allow_domains`는 **WebFetch/bash 네트워크 egress를 호스트 허용목록으로 제한**한다(예: `["api.github.com"]`); 비우면 호스트 제한 없음. 셋 다 전역·프로젝트 config에서 덮어쓰지 않고 **누적(append)**되며, 플러그인의 `set_config_key`가 절대 건드릴 수 없는 고정 deny-list에 있다(EXTENDING §플러그인).

설정 파일 `<config>/config.toml` (macOS `~/Library/Application Support/magi`, Linux `~/.config/magi`):
```toml
model = "gpt-oss:120b-cloud"   # 기본: Ollama 무료 클라우드(ollama signin). 로컬은 "qwen3-coder:30b"
base_url = "http://localhost:11434/v1"
permission = "ask"
experience_dir = "/path/to/team/experience"   # 공유 두뇌(git repo면 팀 공유)

[routing]                  # 에이전트별 라우팅 (프로파일 이름 또는 모델명)
explore = "fast"           # → [llm.profiles.fast] (다른 엔드포인트/키)
planner = "fast"
coder   = "qwen3-coder:30b"  # 기본 백엔드에서 모델만

[llm.profiles.fast]        # 이름붙인 백엔드 (엔드포인트/키/모델/헤더, ${ENV} 확장)
base_url = "https://fast.gateway/v1"
api_key  = "${FAST_KEY}"
model    = "gpt-oss:20b"
[llm.profiles.fast.headers]
X-CLIENT-API-KEY = "${FAST_CLIENT_KEY}"

[orchestration]            # 선제 절차 플래너(기본 on): 요청을 순서 절차로 분해 → step별 전략
planner = true             # (solo|parallel|scout|delegate|refine); scout는 목록 확보 후 각 항목 병렬,
                           # delegate는 독립 write sub-task를 자식에 위임, refine은 의존 sub-goal을 in-context 재귀.
                           # 멀티스텝 계획이면 실행 전 council이 계획을 감사한다: CRITICAL 결함만 차단(재-플랜)하고
                           # warn/info 노트는 비차단 조언으로 수용(실행자에 주입). 완료기준(산출물·테스트 지침)도
                           # 도출해 그 턴의 종료 계약으로 삼는다.
                           # plan_absorb = true → 플래너 1회 추가 패스로 조언을 계획에 접어 넣음(기본 off)
subagent_timeout = "5m"    # 서브에이전트 시도당 하드캡 base; 실효 캡은 관측된 모델 속도에 따라
                           # base/2–base×3으로 탄력 조정. env MAGI_SUBAGENT_TIMEOUT이 우선; /subagent로 라이브 변경.

[mcp.filesystem]           # MCP 서버 (stdio 또는 url=로 HTTP)
command = "npx"
args = ["-y", "@modelcontextprotocol/server-filesystem", "."]

[[hooks]]                  # 라이프사이클 훅 (아래 §하네스 참고)
event = "Stop"             # 턴 종료 직전
command = "go test ./... >/dev/null || echo 'tests failing' >&2"

[council]                  # 합의 종료 게이트(D14): 끄면 모델이 멈출 때 그냥 종료(기본). 켜면 council이 done/continue 투표
enabled    = true
rule       = "majority"    # unanimous | majority | quorum:2 | weighted:0.6 | veto:Balthasar
max_rounds = 3             # 라운드 상한(무진전·취소 안전장치와 함께 무한루프 방지)
preset     = "full"        # "light" = 검증 위원 1인·1라운드 (인터랙티브 지연 절감; member/max_rounds 명시가 우선)
# [[council.member]]       # 생략 시 MAGI 기본 3인 사용
# name = "Melchior"; lens = "correctness"   # lens: correctness|verification|completeness

[theme.dark]               # 컬러 테마 오버라이드 (모드별). 미지정 role은 NERV/MAGI 기본값 유지
primary = "#FF7A1A"        # role: primary·accent·muted·outline·error·success·
accent  = "#5CD8E6"        #       surface·primaryContainer·outlineVariant·warn
[theme.light]
primary = "#B45309"
```
> 위 `[routing]`/`[llm.profiles.*]`/`model`은 **`/route` 에디터로도 편집**되며 이 파일에 저장된다.
> **합의 council(D14, 시그니처 · 기본 on)**: 모델이 턴을 끝내려 할 때 바로 종료하지 않고, **3인 council(Melchior·Balthasar·Casper)**이 과제(목표)·에이전트 보고·툴 결과·**이번 턴 에이전트 자신의 편집**을 보고 done/continue를 투표한다. (편집은 에이전트의 write/edit 툴로부터 재구성된다 — 파일별 before→after diff로, git과 무관하고 정확히 귀속되므로 사람/외부/bash 변경이 에이전트 공로로 잘못 잡히지 않는다; 큰 변경은 한 줄 요약으로 접힘. 상세 모달은 이를 `◆ path` 헤더와 색칠된 +/- 줄로 렌더한다.) 위원은 자기 렌즈로 **구체적 결함**을 짚을 때만 continue, 만족하면 done, 판단할 증거가 없으면 **abstain**(불확실하다고 반사적으로 continue하지 않음). 툴을 전혀 안 쓴 **순수 대화 턴(인사·질문)은 council을 소집하지 않는다**. continue면 합쳐진 피드백을 주입하고 루프를 이어간다 = "종료 판정을 단일 모델에서 빼앗는다". 모든 반려 주입문 끝에는 **작업물 보존 지침**이 붙는다 — 부족한 것은 *증거*이지 재구축이 아니다: 기존 작업물을 삭제·재생성하지 말고, 현재 상태에 대해 요구된 증거를 만드는 최소 행동만 하라(반려당하면 완성물을 밀고 처음부터 다시 만드는 재시작 반사를 차단한다). `rule`로 합의 방식, `max_rounds`로 라운드 상한(무진전/취소 안전장치 포함); **마지막 허용 라운드의 반려는 피드백을 한 번 더 주입하는 대신 즉시 강제 UNVERIFIED로 착지**한다 — 남은 벽시계를 태우다 타임아웃 킬로 실행 중인 산출물 프로세스까지 죽이는 대신, 파일과 프로세스를 세워둔 채 검증으로 넘어간다. `criteria=true`면 과제에서 **완료기준을 턴당 1회 도출**(LLM 1회 추가)해 council 계약으로 삼아 판정이 더 날카로워진다. **기본 켜짐**이며 매 종료 시점에 LLM 라운드가 추가되므로 끄려면 `[council] enabled=false`. 워크플로 모드에선 비활성(파이프라인 자체 verify 게이트 사용). 각 `[[council.member]]`에 `provider`(=[llm.profiles.*] 백엔드)·`model`을 줘 **위원마다 다른 모델(싸구려+강한 혼합)**로 심의시킬 수 있다(미지정 시 세션 모델/기본 백엔드). TUI에선 심의가 **헤더 칩**(`⚖ council rN: <위원>`)과 **트랜스크립트 라인**(소집 / 위원별 **압축 한 줄**(위원 색 `●` + 이름 + 판정 아이콘 — 종료: ✓done·✗reject·∅abstain; 플랜 감사는 심각도별 티어: ✓approve·✎advise(warn)··note(info)·↻revise(critical, 차단)·∅abstain) / 판정·집계)으로 라이브 표시된다. 위원 줄을 **클릭하면 상세 모달**(렌즈·근거·피드백·확신도)이 떠 각 위원이 왜 그렇게 투표했는지 볼 수 있다(esc/클릭으로 닫기). 위원별 색은 테마(`[theme.dark] melchior/balthasar/casper`)로 바꿀 수 있다. `verify = "<명령>"`(단축) 또는 `[[council.signal]]`(name/command, 예: test·lint·typecheck **여러 개**)를 주면(opt-in) 매 라운드 그 명령들을 돌려 **결과를 council 증거로** 넣는다(분위기가 아니라 증거로 판정 = 거짓성공 차단).
> 컬러 테마는 `[theme.dark]`/`[theme.light]`에서 role별로 외부 정의 가능(기본=NERV/MAGI). `--theme`로 모드(auto/dark/light) 선택.
> 첫 실행 시 주석 달린 기본 `config.toml`이 자동 생성된다(있으면 안 건드림).
> **잘못된 config는 절대 조용히 무시되지 않는다**: 전역 `config.toml` 파싱이 실패하면(예: 최상위 키 중복) magi는 파스 에러를 출력하고 시작을 거부한다 — 모델·플러그인·기타 모든 설정을 떨굴 빈 config로 폴백하지 않는다. 잘못된 **프로젝트** `.magi/config.toml`은 warn 후 건너뜀(유효한 전역 config는 여전히 적용). 알 수 없는 *키*(오타)는 warn일 뿐 시작을 막지 않는다.

> **루프 & 실행 안전 가드(결정론적, 항상 켜짐)**: council 외에도 약한 모델이 혼자 빠지는 실패 모드를 잡는 값싼 결정론 가드가 있다. **자기회귀 검사** — 턴 내 각 파일 상태를 추적, 편집이 이번 턴에 이미 있던 상태로 되돌아가면("고쳤다가 슬쩍 되돌리는" 함정) 중립 노트를 붙여 재확인(권고, 차단 안 함, 파일당 1회). **비대화형 명령 실행** — 모든 `bash`는 제어 터미널 없이 stdin 닫고 실행 → 프롬프트를 띄우려는 명령(git 자격증명·ssh 호스트키·apt·pager)이 타임아웃까지 멈추지 않고 **즉시 실패**. **루프 가드** — 진전 없는 동일 툴 호출이 소량 상한을 넘으면 거부(직전 결과 되돌려줌); 계속 헛돌면 먼저 **교정 재-그라운딩 1회**(원 과제 재독·접근 변경) — 이 넛지에는 **완료 탈출구**가 있다: 작업이 정말 끝났으면 **툴 호출 없이** 답만 하고 턴을 끝내라(그것이 검증을 트리거한다; 확인 명령 하나 더는 진전이 아니다), 그리고 **활동을 보여주려고 완성된 작업물을 삭제·재빌드하지 마라**. 그래도 지속되면 예산을 다 태우지 않고 우아하게 중단. **스톨 강제중단** — 교정 넛지 후에도 여러 스텝 진전이 없으면 천장까지 갈지 않고 우아하게 중단(`stall_guard`); `bash` 쓰기는 진전으로 인정해 느린 빌드를 스톨로 오인하지 않는다. **빈-종료 넛지** — 답변 텍스트 **없이** 끝나는 턴은 사용자가 읽을 게 아무것도 없다. harmony 약한 모델이 analysis 채널만 뱉고 멈추거나("reasoning-only stop"), 툴을 쓴 뒤 마지막 스텝에서 침묵할 때 생기는데 — 후자는 council 게이트가 그 툴 작업을 보고 "done"으로 표결해 딜리버러블도 UNVERIFIED 플래그도 없이 끝날 수 있다. 이 턴은 툴 사용 여부와 무관하게 **1회** 넛지해 실제 결과를 쓰게 한다(서브에이전트가 `report`를 부르도록 받는 넛지와 동일); 그래도 빈 재시도면 정상 종료하므로 루프가 되지 않는다.

> **Concern ledger & 체크포인트-우선 계획(다단계 작업).** council이 초기에 제기한 구조적 우려 — "auth 변경에 테스트 없음", "이 마이그레이션은 되돌릴 수 없음" — 를 세 턴 뒤 무관한 편집에 파묻혀 잊으면 안 된다. magi는 그런 우려를 **턴을 가로질러 지속되는 원장(ledger)**에 담아 플래너에 재노출하고, **체크포인트-우선 게이트**로 계획이 완료를 선언하기 전에 열린 우려를 먼저 처리하게 한다 — 알려진 미해결 리스크를 테이블에 둔 채 "done"으로 끝내지 못하게. 전체 맥락(원 요청·전체 계획·모든 서브에이전트 결과)은 오케스트레이터만 쥐므로, **우려를 은퇴시키는 것도 오케스트레이터만** `resolveconcern` 툴로, 그리고 진짜로 해결된 뒤에만 가능하다 — 실행자가 제 리스크를 슬쩍 지울 수 없다.

> **압박하 반-fabrication.** 이 가드들이 겨냥하는 실패 모드는, 줄어드는 스텝/시간 예산이나 막다른 길에 몰린 약한 모델이 막혔다고 인정하는 대신 그럴듯한 결과(파일 내용·테스트 결과·집계값)를 **지어내는** 것이다. 시스템 프롬프트는 스톡/예산 압박하에서 결과나 증거를 지어내는 것을 명시적으로 금지하고 정직한 부분 상태를 보고하게 한다; 결정론 **데이터 툴**(§5)은 집계를 실제로 계산할 길을 줘 추측할 필요를 없앤다; council은 모델이 끝냈다는 주장이 아니라 **갓 나온 실행 증거와 시그널-명령 결과**(아래)로 판정한다 — 그래서 자신만만하지만 뒷받침 없는 "done"은 잡히고, 정직한 "Y 때문에 X를 못 했다"는 받아들여진다. 정직한 실패는 올바른 결과이고, 지어낸 성공이 버그다.

### 시간 & 스텝 예산

매 스텝, magi는 에이전트 컨텍스트에 (캐시되는 시스템 프롬프트가 아니라 — 접두사 캐시 안정) **예산 라인**을 덧붙여 모델이 스스로 페이스 조절하게 한다:

- **스텝 예산** — `You are on step N of at most M`. 천장 `M`(기본 **240**, `MaxSteps`)은 **할당량이 아니라 하드 천장**으로 제시된다: 일찍 끝내는 게 더 좋고, 오직 *서술만* 하는 명령(“완료”, 상태 요약 echo)에 스텝을 쓰는 건 명시적으로 금지 — 대신 응답 텍스트로 말하라. 작은 단계 예산(내부 summarize=3 등)은 라인 자체를 건너뛴다.
- **소프트 플래너 추정** — 플래너가 돌았으면 `estimated_steps`를 *"대략 K 스텝으로 추정 — 한계가 아니라 페이스 참고"*로 보여준다. **권고일 뿐**: 약한 모델 추정은 흔히 틀리고 잘못된 하드 캡은 진짜 작업을 자르므로, 240 천장이 유일한 정지선이다. 추정을 크게 넘어서면 멈춰 재점검하라고 알린다.
- **자기측정 경과** — 턴이 1분을 넘기면 *"You have been working for 11m of wall-clock time so far"*를 덧붙인다. 이는 (턴 시작부터 잰) magi 자신의 **스톱워치**이지 외부 채점기 정보가 아니어서 어디서든 공정하며, 느린 반복(컴파일 10회 재시도)을 알아채고 접근을 바꾸게 한다.
- **`--time-budget`(기본 꺼짐)** — 사용자가 지정한 소프트 마감(예: `--time-budget 20m`). 켜면 예산과 남은 시간을 표시하고, 경과가 넘어가면 *"budget EXCEEDED — 가장 작은 정직한 결과를 즉시 내라"*로 바뀐다. **권고이지 하드 정지 아님**이며, 결과를 apples-to-apples로 유지하려 **리더보드/비교 런에선 끈다**.

**council 비용 캡** — council도 자기 벽시계를 본다. 심의가 턴의 과도한 몫(최소 60초 **그리고** 턴 경과의 1/4 이상)을 소비하면, 추가 라운드는 이득보다 비용이 커지므로 라운드 2+를 건너뛰고 3인 라운드를 또 치르는 대신 **UNVERIFIED**로 표기해 끝낸다. 첫 라운드는 항상 돈다.

### 하네스 (기본 켜짐)

설정 없이도 "이해→계획→구현→검증→요약" 절차가 자연히 적용된다. 두 층으로 구성:

1. **운영 가이드 프롬프트(항상)** — 다단계 작업이면 `todowrite`로 계획 후 하나씩, 편집 후 빌드/테스트로 검증, 깨진 상태로 끝내지 않고, 마지막에 변경 요약. 사용법을 몰라도 그냥 대화만 하면 적용됨.
2. **내장 훅(항상, `--no-harness`로 끔)** — 파일 편집 직후 자동 포맷(gofmt) + 언어 진단(gofmt -e / go vet / py_compile)을 돌려 에러를 모델에 되먹임 → 자가 수정.

**팀 공유**: 프로젝트 루트의 `.magi/config.toml`을 커밋하면 워크플로가 레포와 함께 이동한다. 전역(`<config>/config.toml`)과 병합되며 `[[hooks]]`는 누적된다.

훅 이벤트:
| event | 시점 | 종료코드 ≠ 0 효과 |
|---|---|---|
| `PreToolUse` | 툴 실행 전 | 툴 **차단**(stderr가 모델에 전달; 경로 보호 등) |
| `PostToolUse` | 파일 편집 후 | 출력을 피드백으로 전달 |
| `Stop` | 턴 종료 직전 | 종료 막고 계속 작업 강제(예: 테스트 통과 요구) |

훅 명령은 셸로 실행되며 `MAGI_TOOL`/`MAGI_PATH` 환경변수 + JSON stdin을 받는다. `match`로 툴 이름 필터(`"*"`=전체).

타이핑 키는 입력창으로만 가고 스크롤은 위 전용 키로만 — 그래서 본문 타이핑(띄어쓰기 포함)이 화면을 스크롤하지 않는다. 위로 스크롤해 읽는 중이면 스트리밍이 화면을 끌어내리지 않는다(바닥에 있을 때만 자동 추적). 트랜스크립트가 넘치면 **헤더 칩**이 스크롤 위치를 보여준다(`⇅ 42% (120/300)`, 위에 있는 동안 새 출력이 오면 `↓ new` — End로 복귀). 그리는 스크롤바는 제거됐다(칩이 대체 — 윈도우 ambiguous-width 정렬 버그 클래스가 함께 소멸).

작업한 턴이 끝나면 **영수증 한 줄**(`▣ turn: 14 steps · 3 file(s) · council r2 · 3m49s`)이 남아 스크롤 없이 턴 비용을 볼 수 있다. 에이전트가 진짜 선택지를 물어야 할 땐 **선택 모달**(`ask_user` 툴)이 뜬다: ↑/↓ 또는 1-9로 고르고 enter, esc는 건너뛰기(모델에겐 "무응답 — 알아서 판단"으로 전달). 질문이 여러 개면 모달이 순서대로 하나씩 뜬다.

**권한 영속(`p` = 프로젝트):** `p`를 고르면 `.magi/config.toml`에 allow 규칙을 써 재시작해도 유지된다. 규칙은 **툴이 허용하는 만큼 좁게** 스코프된다: 대부분 툴은 `tool(**)`로 영속되지만, `bash`는 승인한 **프로그램 이름**만 영속한다 — `curl https://x`를 허용하면 `bash(**)`가 아니라 `bash(curl:*)`로 기록 — 그래서 한 번 허용이 이후 모든 명령을 몰래 사전승인하지 않는다. 셸 메타문자(파이프·리다이렉트)로 시작하는 명령은 고정할 프로그램이 없어 과도승인 대신 세션 한정으로 남는다. 허용된 프로그램이라도 파괴/egress 스캐너는 위험한 호출에 여전히 재확인한다.

**Ambiguous-width 문자(주로 윈도우 관련):** 기동 시 magi가 터미널을 한 번 프로브해 셀-폭 측정을 자동으로 맞춘다(윈도우는 Console-API 커서 델타, 그 외는 커서 위치 질의). 프로브를 못 돌리거나(예: 리다이렉트된 stdio) 잘못 추정하면 `MAGI_AMBIGUOUS_WIDTH=wide`(또는 `narrow`)로 강제하고, `MAGI_WIDTH_PROBE=0`은 프로브를 끈다. 표준 `RUNEWIDTH_EASTASIAN=1`도 존중된다.

### 마우스 / 텍스트 복사 (모드 없음)
휠 스크롤·드래그 선택·클릭 포커스가 **모드 전환 없이** 모두 된다 — 선택/복사를 앱이 직접 처리하기 때문이다. **드래그하면 그 범위가 하이라이트되고(문자/셀 단위, 줄 부분 선택 가능), 떼는 순간 클립보드로 복사**된다(OS 클립보드 `pbcopy`/`wl-copy`/`xclip` + OSC52 둘 다 시도). 드래그 중 휠 스크롤도 된다(선택은 콘텐츠 위치에 고정되어 스크롤해도 유지).

### 트랜스크립트 렌더링
- **edit/write는 색칠된 diff**로 보인다 — 구문 강조(파일 확장자로 언어 감지) + 줄번호 거터, 추가/삭제는 `+`/`-`로.
- **bash·read·grep·glob·list·webfetch·websearch 결과는 접힌 블록**으로 본문에 표시(read는 줄번호+구문 강조). 길면 `… +N more` 푸터가 붙고 **블록을 클릭하면 펼침**.
- **카운슬 거부 사유**는 한 줄 판정 아래에 줄바꿈되어 표시(전체 사유는 위원 클릭으로 상세 모달).
- **카운슬 판정 대기 표시**: 라운드가 열려 심의가 진행되는 동안 하단 상태줄이 스피너와 함께 무엇을 기다리는지 한 줄로 고정 표시한다 — 플랜 감사면 `⚖ 플랜 감사 판정 대기 중…`, 심의/합의면 `⚖ 카운슬 심의 판정 대기 중…`. 침묵처럼 보이던 대기 구간이 진행 중임을 알려준다.
- **검수 전 보고서 접기**: "검수해줘"류 요청은 보고서 → council 리뷰 → 개정 보고서 순으로 흐르는데, 리뷰 라운드가 continue로 판정하면 **개정본이 실제로 도착하는 순간** 원래(검수 전) 보고서를 한 줄 스텁(`≡ (검수 전 보고서 — 접힘, 아래 최종본 참고)`)으로 접어 최종본만 남긴다. 접기는 개정본 도착 시점으로 미뤄지므로 리뷰가 중단·오류로 끝나면 원본이 그대로 보존되고, 개정본이 원본과 사실상 동일하면 원본을 건드리지 않는다(깜빡였다 다시 뜨는 현상 방지).

### 상태 패널 (포스트잇)
계획(todos)·서브에이전트·컨텍스트가 있을 때 **우상단에 둥근 윤곽선 박스(포스트잇)**로 떠서 보인다(없으면 숨김). 트랜스크립트는 전체 폭을 쓰고 박스는 그 위에 겹쳐 그려지며(하단 정렬이라 보통 빈 공간에 뜸), **박스 왼쪽 모서리를 드래그하면 폭이 조절**된다. 서브에이전트 줄을 클릭하면 그 에이전트로 줌인. 박스 테두리는 터미널이 실제로 그리는 셀 폭 기준으로 조립되므로, todo·서브에이전트 줄에 🚀 같은 이모지가 있어도(터미널이 1칸으로 그리든 2칸으로 그리든) 우변 `│`가 어긋나지 않는다 — 기동 시 이모지 폭을 한 번 프로브해 보정하며, `MAGI_EMOJI_WIDTH`로 강제하거나 `MAGI_WIDTH_PROBE=0`으로 끌 수 있다.

### 테마
시작 시 터미널 배경색으로 다크/라이트를 감지하고, **실행 중 OS 테마를 바꾸면 몇 초 안에 따라 전환**된다(배경색을 주기적으로 재질의). `--theme`(또는 `MAGI_THEME`)로 `auto`/`dark`/`light` 강제.

### 작업 중 끼어들기 (steering)
작업(턴)이 도는 동안에도 입력은 살아 있다 — 계속 타이핑할 수 있고(한글 IME 포함 인라인 조합), Enter를 누르면 메시지가 **진행 중인 턴에 즉시 주입**된다. 메인 에이전트는 **다음 스텝**에서 그 메시지를 보고 반영한다 — 큐에 쌓였다가 턴이 다 끝나야 나오는 게 아니라, 실행 중인 에이전트를 그 자리에서 조종(steer)하는 것. 메시지는 트랜스크립트에 바로 표시된다.
메인 에이전트는 `task`로 위임한 서브에이전트를 **백그라운드(사이드카)로 돌리고 즉시 반환**하므로 블록되지 않는다 — steer 메시지에 바로 반응한다.
메인이 **서브에이전트 보고를 기다리는 것 말고 할 일이 없는 동안**에도, 보낸 메시지는 그 자리에서 처리된다 — 서브가 끝날 때까지 묵히지 않는다. 잡담엔 짧게 답하고, steer는 진행 중 작업을 실제로 바꾼다: "`docs/` 아래만 봐"는 이제 무의미해진 읽기 작업을 취소하고 방향을 전환, "그거 다음에 README도 써줘"는 추가 스텝을 현재 작업에 접어 넣고, 모호한 steer는 먼저 짧게 되물어 구체화한 뒤 반영한다. (대기 중엔 변경을 *신호*만 하고, 실제 재-플랜/재-디스패치는 다음 스텝에서 수행한다.) steer가 실제로 반영되면 트랜스크립트에 "Steer applied …" 감사 줄이 남아 방향 전환/취소가 실행됐음을 확인할 수 있다 — 말로만 수긍하고 넘어가지 않는다. 방향을 바꾸면 재-플랜은 (원래 목표에 steer 제약을 접어 넣은) *채택된* 작업 기준으로 다시 분해된다.
현재 작업이 끝나야 답할 수 있는 질의를 넣으면, 그 질의는 큐에 들어가 **입력한 자리에 그대로 보이다가**, 답변이 준비되면 **답변 바로 위로 재배열**되어 질의–답변이 한 쌍으로 붙는다(세션을 다시 열어도 중복 없이 짝이 유지된다).
작업 중 슬래시 커맨드: 읽기/UI 전용(`/help`·`/route`(=`/model`=`/agents`)·`/tools`·`/sessions`·`/diff`·`/permission`)은 실행되고, 세션을 바꾸는 것(`/resume`·`/rewind`·`/clear`·`/init`·`/ultra`·`/compact`)은 작업 중 거부된다.

### 멀티에이전트 라이브 뷰 (split-pane)
서브에이전트가 떠 있는 동안 메인 트랜스크립트 아래에 **각 서브에이전트의 라이브 패널**이 타일로 표시된다(각 child 세션을 실시간 구독). 서브에이전트마다 **고유 색상**(M3 톤 팔레트)이 배정돼 패널 보더·헤더 배지·트랜스크립트의 `⚙ task → <이름>` 하이라이트에 일관 적용된다.
- `Tab`(또는 패널 클릭)으로 포커스 이동 → 포커스 패널엔 그 색상의 **포커스 링**.
- `Ctrl+O`(또는 포커스 패널 재클릭)로 **줌인** → 그 서브에이전트의 전체 트랜스크립트를 상세 관찰. 줌 진입 시 맨 아래(최신/결론)로 이동. 상단 **breadcrumb `‹ back  ✦ magi › coder`** 를 **클릭**(또는 `Esc`)하면 복귀. 상세 본문의 어시스턴트 라인은 `magi`가 아니라 **그 에이전트 이름(색상)**으로 표시.
- **색상이 식별자**: 메인이 같은 종류 에이전트에게 여러 작업을 시키면 **작업(세션)마다 독립 패널**이 뜬다. 같은 역할은 **색(hue)은 동일**(트랜스크립트 `task → coder` 하이라이트와 일치), **2번째·3번째는 명도**로 구분된다.
- **패널은 역할 기준**으로 관리된다: 같은 역할(coder 등)이 재시작/재위임돼도 **새 창을 만들지 않고 기존 창을 재사용**한다.
- **수명**: 작업 중엔 타일로 크게, **턴이 끝나면 한 줄짜리 compact로 접힌다**(여전히 `Ctrl+O`로 열어볼 수 있음). 다음 메시지를 보내면 사라지고, 나중에 `/resume`로 복원된다.

### 우측 상태 패널
계획·진행 상황이 있을 때 화면 우측에 **상태 패널**이 뜬다(없으면 숨김). **좌측 경계를 마우스로 드래그**해 폭을 조절할 수 있다(기본 44칸). 섹션:
- **Plan** — 현재 todo 체크리스트 + 진행도(`done/total`): 완료 ✓, 진행 중 ◐, 대기 ☐, **취소 ✗**(턴이 중단/정지돼 못 끝낸 단계). 진행 표시는 결정론적 신호로 구동되어(플래너가 실행한 단계를 체크오프하고, 나머지는 턴 종료 시 확정) 모델이 `todowrite`를 안 불러도 갱신된다. 실시간 갱신.
- **Subagents** — 활성 서브에이전트 목록(색상·상태·토탈 `소요시간·↑↓토큰`). **항목 클릭 → 그 서브에이전트 상세(zoom) 진입**(pane 클릭과 동일).
- **Context** — 컨텍스트 토큰 사용 막대.
서브에이전트 줌(상세) 화면에서도 동일 레이아웃으로 뜨며, 이때 Plan은 **그 서브에이전트의 todo**를 보여준다.

### 붙여넣기 접기
여러 줄(또는 200자 초과) 붙여넣기는 **입력창에서만** `[#N pasted L lines]` 칩으로 접힌다(입력창이 좁으므로). 전송하면 **트랜스크립트(메인)에는 전체 내용이 그대로** 표시되고, 에이전트에도 전체가 전달된다. (↑ 히스토리 recall은 칩 형태로 불러와 입력창을 어지럽히지 않는다.)

### `@` 파일 멘션
메시지에 `@경로/파일`을 넣으면 해당 파일 내용이 에이전트 컨텍스트에 첨부된다.

### 헤더 표시
`model <id> · ctx <%>` + **퍼미션 칩(색상 구분)** + 서브에이전트 실행 중 `⛐ N: explore, coder×2` 배지(이름·색상).
- 퍼미션 칩 색: `ask`=amber(안전) · `auto`=cyan(편집 자동) · `allow`=yellow(주의) · `deny`=red(차단).
- 다단계 계획이 실행 중이면 헤더가 **활성 플랜 스텝**(절차 플래너 체크리스트의 현재 항목)도 표시해, 지금 에이전트가 무엇을 하는지 상태 패널을 열지 않고도 보인다.

### 세션 재개 (`/resume`)
`/resume`(인자 없이) → **인터랙티브 피커**: 각 세션의 시간 + 첫 메시지 요약을 보여주고 `↑/↓`로 선택, `Enter`로 재개, `Esc`로 취소. `/resume N`으로 바로 전환도 가능.
- 목록에는 **사용자 세션만** 나온다(서브에이전트가 만든 child 세션은 숨김).
- 부모 세션을 재개하면 그 세션의 **서브에이전트들이 완료 패널로 복원**되어 `Tab`/`Ctrl+O`로 다시 들여다볼 수 있다(라이브 spawn 이벤트는 휘발이라 디스크의 child 세션에서 복원).

### TUI 동작 체크리스트 (스크린샷)
검증 중인 인터랙션 동작 모음 — 각 항목 아래에 스크린샷을 붙이기 좋게 정리.

1. **작업 중 입력 + 한글 IME** — 턴이 도는 동안에도 입력창에 인라인으로 타이핑/조합됨(커서가 입력 위치에).
2. **Steering(작업 중 끼어들기)** — 작업 중 Enter로 보낸 메시지가 진행 중 턴에 즉시 주입, 다음 스텝에서 메인이 반영. 서브에이전트가 돌아도 메인은 바로 반응.
3. **멀티에이전트 split-pane** — 서브에이전트별 라이브 패널이 타일로, **에이전트마다 고유 색상**. `⚙ task → <이름>`도 같은 색.
4. **포커스 / 줌** — `Tab`(또는 패널 클릭, 마우스 ON일 때)으로 포커스 이동(색상 포커스 링), `Ctrl+O`로 전체화면 상세(브레드크럼·구분선·좌측 바가 그 색), `Esc` 복귀.
5. **서브에이전트 개별 인터럽트** — 실행 중 서브에이전트에 포커스 후 `Esc` → 그 서브에이전트만 중단(나머지·메인은 계속).
6. **슈퍼바이저(사이드카)** — 무응답/스톨 시 자동 재시작(헤더에 `restarting…`), 타임아웃·재시작 소진 시 ERROR 결과 주입. 하나가 죽어도 시스템은 안 멈춤.
7. **붙여넣기 접기** — 멀티라인 붙여넣기 → 입력창엔 `[#N pasted L lines]` 칩, 전송 시 메인엔 전체 내용.
8. **스크롤 위치 유지** — 바닥에 있으면 스트리밍/패널 추가·제거·**터미널 리사이즈** 후에도 바닥 유지, 위로 스크롤해 읽는 중이면 안 끌려감.
9. **입력 히스토리** — `↑/↓`로 이전 프롬프트 불러오기(세션 재개 시 이전 턴도 포함), `Tab`으로 접두사 자동완성.
10. **마우스/복사 (모드 없음)** — 휠 스크롤·드래그 선택+복사·클릭 포커스가 모드 전환 없이 모두 됨(앱이 선택을 직접 처리). 드래그 중 휠 스크롤도 됨.
11. **시작 시 화면 정리** — 실행 시 터미널을 한 번 clear하고 깨끗한 화면에서 시작.

## 6. 멀티에이전트

`task` 툴로 서브에이전트에 위임한다. 기본 에이전트:
- **explore** — 읽기전용 코드 탐색
- **reviewer** — 코드 리뷰(읽기전용)
- **coder** — 구현(read/write/edit/multiedit/grep/glob/list/bash)

제한(D7): 깊이 3 · 동시 8 · 누적 50. `[routing]`으로 에이전트별 모델 지정.

### 사이드카 실행 모델 (메인 = UI 스레드)
최상위 오케스트레이터가 `task`로 위임하면, 서브에이전트는 **백그라운드 사이드카**로 돌고 `task`는 즉시 반환한다. 메인 에이전트는 블록되지 않고(=UI 스레드처럼 비어 있어) **사용자 입력에 바로 반응**하며, 각 서브에이전트 결과는 **끝나는 대로** 메인 세션에 주입돼 **증분 처리**된다. 주입된 결과에는 **아직 실행 중인 서브에이전트 수**가 함께 붙어, 오케스트레이터가 남은 것을 기다릴지(재위임 말고)·결과 기반으로 **새 후속 작업을 위임**할지·종합할지 스스로 판단한다. 무거운 작업은 위임하고 가벼운 건 인라인으로 처리한다.

각 사이드카에는 **슈퍼바이저**가 붙어 헬스체크한다. 오케스트레이터는 위임한 작업이 얼마나 걸릴지 미리 알 수 없으므로, 두 워치독은 의도적으로 비대칭이다 — 촘촘한 liveness 체크 + *넉넉한* 벽시계 백스톱:
- **스톨 감지**(`SubagentStall`, 기본 4분 무활동) — 1차 liveness 가드로, 완전히 멈춘 자식(이벤트가 전혀 없는 상태)을 잡는다. **활동 기반**을 유지한다: 스트리밍 reasoning/text 델타를 포함한 어떤 이벤트든 타이머를 재무장하므로, 느린 단일 생성은 절대 오사살되지 않는다. 또한 **툴 실행 중에는 억제된다** — 조용한 장기 툴(예: 수 분짜리 `bash` 빌드)은 반환 전까지 아무 이벤트도 내지 않으므로, 행으로 오인하지 않고 툴 자체 타임아웃에 맡긴다.
- **하드 타임아웃**(`[orchestration] subagent_timeout`, 기본 5분/시도 = **base**) — 이벤트를 계속 발생시키며 공전하는 서브에이전트(환각 타깃 조사, 부모와의 질문 핑퐁)는 스톨 워치독에 절대 안 걸리므로, 하드캡이 유일한 상한이다. 캡이 후하면 공전 explorer 하나가 부모의 벽시계 예산 전체를 잡아먹는다. 실효 캡은 **탄력적**이다: 모델별 최근 LLM 왕복 시간을 추적해 시도당 ~6왕복 분량을 배정하고 base/2–base×3으로 클램프한다 — 느린 모델의 정당한 긴 생성이 중간에 잘리지 않고, 빠른 모델에는 공전할 여유를 주지 않는다. base는 설정(`subagent_timeout = "5m"`), 런 단위 `MAGI_SUBAGENT_TIMEOUT`, 또는 **`/subagent <기간>`**으로 라이브 조정(맨 `/subagent`는 base+현재 실효 캡 표시). 만료는 무조건 킬이 아니라 **판단 리스**다: 오케스트레이터의 모델이 자식의 최근 툴콜 결정론적 다이제스트(연속 중복은 반복 횟수로 압축)를 보고 EXTEND(진짜 진전 → base/2 추가) 또는 KILL(공전 → 재시작 정책 적용)을 판정한다. 판정은 모든 축에서 바운드된다 — 만료 시점에만 발화(캡 안에 끝나는 보통 시도는 비용 0), 판정 호출 자체가 시간 제한, 에러·모호한 답은 KILL로 fail-safe, 그리고 판정과 무관한 **절대 백스톱 base×3** — 판정자가 속아도 피해는 백스톱까지다. `MAGI_SUBAGENT_JUDGE=off`로 판정을 끄면 만료 즉시 킬(순수 엘라스틱 캡 동작). 정상적인 집중 서브에이전트 작업은 5분 안에 충분히 끝난다; 진짜 긴 위임 작업은 캡을 늘릴 게 아니라 쪼개야 한다. 플래너의 백그라운드 explorer에는 더 빡빡한 **spawn 전체 바운드**(3분, 재시작 포함)가 걸린다: 만료는 재시도 없이 종결되므로, 공전하는 explorer의 실패 결과가 캡×재시작 예산만큼 부모를 붙잡는 대신 몇 분 안에 파킹된 부모에게 도착한다.
- **자동 재시작**(`SubagentMaxRestarts`, 기본 2회) — 스톨/타임아웃/일시 오류 시 재시도(같은 역할 패널 재사용), 소진 시 ERROR 결과 주입. 하나가 죽어도 나머지·메인은 계속.

(중첩 서브에이전트는 동기 위임으로 동작 — 백그라운드는 최상위에서만.)

### 에스컬레이션 (서브에이전트 → 오케스트레이터 `ask`)
서브에이전트는 실행 중 막히면 **`ask` 툴로 오케스트레이터에게 물어보고 그 자리에서 답을 받는다**(블록 후 재개). 오케스트레이터는 **전체 맥락**(사용자 원 요청·전체 계획·다른 서브에이전트 결과)을 갖고 있어: 의도 명확화·결정, 경로/제약 제공, **사용자에게 재질문 relay**, **다른 서브에이전트와 조율**(peer 질문도 오케스트레이터 경유 — 그래야 맥락이 한곳에 모임)을 해줄 수 있다. `ask` 툴 설명에 "오케스트레이터가 해줄 수 있는 것"이 명시돼 서브에이전트가 무엇을 요청할지 안다. 한 번에 하나의 ask만 처리(직렬화), 2분 타임아웃.

## 7. 메모리 & 컨텍스트

- **AGENTS.md**: 작업 디렉터리(+`.magi/AGENTS.md`, 전역 `<config>/AGENTS.md`)의 내용이
  시스템 프롬프트에 주입되어 **압축돼도 보존**된다. `/init`로 자동 생성.
- **자동 압축(재수화 가능)**: 실제 토큰 수가 모델 윈도우 80% 초과 시 오래된 턴 요약(최근 보존). 압축된 구간은 **topic shard로 결정론적 인덱싱**되어 원문이 사라지지 않는다 — `recall_context("<topic>")`(파일 경로가 잘 맞음)로 **원문 그대로** 재수화(토픽당 1회·턴 예산·크기 캡). 여기에 매 스텝, 현재 작업(최근 사용자 프롬프트 + 마지막 어시스턴트 메시지)과 어휘적으로 맞는 압축-이탈 토픽 최대 3개가 **힌트로 자동 주입**되어(“관련 과거 컨텍스트 — recall_context로”) 모델이 recall을 떠올리지 못해도 잃은 걸 가리켜 준다. 매칭은 **BM25-lite IDF**로 순위를 매긴다: 여러 shard에 흔한 토큰(`the`, `handler`)보다 한 구역을 콕 집는 희귀 토큰(`dehydration`, `heap.go`)이 앞선다 — 그래서 현재 스텝이 실제로 필요한 구역을 가리킨다. 순수 어휘/결정론(임베딩 의존 없음)이며 힌트는 *가리키기*만 하고 원문은 여전히 `recall_context`로 당겨온다. 윈도우는 **모델별**(에이전트마다 다른 모델→다른 윈도우)로, 시드에 없으면 백엔드에 **질의**(vLLM `max_model_len`·LiteLLM `/model/info`·Ollama `/api/show`) — 초기 모델은 시작 시, 그 외 모델은 **처음 쓰일 때 지연 조회**(`/route` 런타임 전환 등). Claude/Gemini는 해당 엔드포인트가 없어 시드 테이블에 의존. 윈도우를 못 구하면 **무제한**(게이지·비율압축 비활성)으로 간주하고, `/context <model> <tokens>`로 수동 지정 가능.
- **공유 두뇌(D13)**: `<config>/experience`(또는 `experience_dir`)의 `memories/`·`skills/`를
  세션 시작 시 회수·주입. `remember` 툴은 `pending/`에 기여(리뷰 후 `memories/`로 이동).
  디렉터리를 git repo로 두고 commit/pull하면 팀 공유.
  단계별 부트스트랩/파일형식/리뷰 절차: [`EXTENDING.md`](EXTENDING.md) §2.

## 8. 스킬

`<config>/skills/*.md` 또는 `<workdir>/.magi/skills/*.md` (첫 줄=설명, 이하=본문), 그리고 **skill-creator 규격** `<workdir>/.claude/skills/<slug>/SKILL.md`(frontmatter `description`=트리거) — Claude Code/OpenCode 도구와 번들 engram 플러그인이 만드는 형식을 변환 없이 읽는다. 스킬 목록은 소스 디렉토리 변경(mtime) 시 갱신되므로, engram이 세션 중 저장한 스킬도 재시작 없이 다음 턴부터 쓸 수 있다.
시스템 프롬프트에 목록이 노출되고, 모델이 `skill` 툴로 본문을 로드해 따른다.

## 9. 플러그인 (Lua)

`<config>/plugins/<name>/` 또는 `<workdir>/.magi/plugins/<name>/`에
`plugin.toml` + `init.lua`. capability: `tool`·`command`(`/login` 같은 슬래시 커맨드)·`context-provider`·`mcp`·`llm-headers`·`analyze`·`experience`(magi.propose_experience — 플러그인이 학습한 교훈/스킬을 D13 공유 스토어 리뷰 큐로 제안). 파일 변경 시 **핫리로드**.
샌드박스(위험 stdlib 차단) + 매니페스트 권한(`fs:read`, `net`, `exec`) 집행.
예제: `plugins/examples/wordcount`.

**관찰자(observer) 플러그인.** 라이프사이클 이벤트(`startup`/`shutdown`/`session_start`) 외에 `magi.on`은 페이로드 테이블을 받는 **관찰 이벤트** 2종을 지원한다: `user_message`(`{session, text}` — 진짜 사용자 프롬프트 제출)와 `turn_finished`(`{session, text, outcome, reason, skills}` — 톱레벨 턴 종료, text는 최종 어시스턴트 답변; `skills`는 이번 턴에 로드된 스킬명 콤마 목록 — 사용 실적 계측용). `outcome`은 턴의 **구조적 판정**이라 관찰자가 성패를 문구에서 추측할 필요가 없다: `verified`(카운슬이 직접 done 합의) · `unverified`(승인 없이 착지 — 딜록/비용캡/라운드캡) · `guard`(루프/스톨 가드 강제정지) · `error` · `done`(판정 없는 평이한 종료); `reason`은 사유. 둘 다 **턴 경로 밖에서 비동기로**(바운드 큐 + 워커 1개) 발화되므로 느린 핸들러가 대화를 지연시키지 않고, 큐가 넘치면 이벤트를 버린다(관찰은 best-effort). 여기에 **`magi.analyze{system=, text=, model=?}`** — 도구 없는 **원샷 사이드카 LLM 호출**(토큰을 쓰므로 capability `analyze` 필요; 시간 제한; model 생략 시 세션 모델) — 과 **`magi.json_decode(s)`**를 조합하면, 플러그인이 대화를 관찰해 구조화 지식(교훈·요약)을 추출하고 `magi.write_file`로 축적한 뒤 `magi.register_context_provider`로 되먹일 수 있다. 번들된 **`plugins/engram`** 자기개선 플러그인(구조적 outcome을 게이트로 교훈/스킬 자동 추출)이 이 3종 조합의 레퍼런스다 — 해당 README 참고.

```toml
# plugin.toml
name = "wordcount"
capabilities = ["tool"]
permissions = ["fs:read:."]
```

**설치 / 업데이트.** git repo로 배포된 플러그인(repo 루트에 `plugin.toml`)은
`magi --plugin-install <git-url> [--plugin-pin <태그/브랜치>]`로 설치하며, `<config>/plugins/`로
clone된다. `magi --update-plugins`(또는 바이너리까지 갱신하는 `--update`)는 git 체크아웃 플러그인마다
fast-forward하며, 로컬 변경이나 remote 없음은 보고 후 건너뜀(덮지 않음). §버전/자동 업데이트 참고.

## 11. 모델 권장

- **gpt-oss:120b-cloud** — **기본**. Ollama 무료 클라우드 티어(`ollama signin`), GPU 불필요. 강력한 범용+코딩.
  무료는 "light usage"(동시 1개·GPU시간 쿼터)라 더 무거운 `qwen3-coder:480b-cloud`는 쿼터를 빨리 먹는다.
- **qwen3-coder:30b** — 가장 강한 **로컬** 코더(24GB GPU). `--model qwen3-coder:30b`로 완전 로컬 실행.
- **gpt-oss:20b** — 가벼운 로컬 대안(추론 표시).
- 작은 모델(llama3.1:8b 등)은 툴 활성 시 함수콜 누설 경향 → 비권장.

**프로바이더 복원력(불안정한 백엔드가 턴을 거의 잃지 않는 이유).** magi는 OpenAI 호환 계층을 best-effort로 다루며, 안전하게 가능한 곳에선 어보트 대신 복구한다:
- **tool-call 파싱** — 로컬/클라우드 모델의 tool-call 변종(JSON/XML/네이티브)을 모두 파싱하고, 백엔드가 `cache_control`을 거부(400/422)하면 투명하게 캐싱 없이 1회 재시도하고 세션 동안 기억한다.
- **일시 장애** — 연결 오류와 429/5xx는 상한 있는 백오프로 재시도(`Retry-After` 준수); 재시도 소진 시 **상태 + 본문**을 표면화해 실패가 진단 가능하며, 맨숭한 "request failed"로 끝나지 않는다.
- **harmony tool-call 오파싱** — Ollama의 gpt-oss harmony 파서는 **모델이 최종 답을 prose로 뱉을 때** 그걸 tool call로 읽으려다 종종 **500**을 낸다(`error parsing tool call: raw=…`). 재시도 사이 요청이 동일해 이 500은 결정론적이므로, magi는 **tools 배열을 벗기고 1회 재시도**한다: tools를 광고하지 않으면 서버가 tool-call 파싱을 건너뛰고 같은 prose를 일반 content로 돌려줘 — 모델이 실제로 낸 답이 턴 하드-어보트 대신 복구된다. 이 정확한 시그니처에만 스코프되어 진짜 장애(genuine outage)는 그대로 에러로 표면화된다.

## 12. 상태 & 범위

**루프 엔지니어링 트랙은 계획이 아니라 출하됐다** — 이 툴의 시그니처이며 이 매뉴얼 전반에 기술돼 있다: 종료 판정을 단일 모델에서 빼앗는 **합의 council**(Melchior·Balthasar·Casper, §3), **Loop map**(`/loop`), 라이브 심의 패널, **rewind/fork/세션-diff**(`/rewind`·`/fork`·`/loopdiff`, §4), **concern ledger**(§3), **재수화 가능 컴팩션**(§4). 마찬가지로 이미 구현됨: **OS 샌드박스**(`--profile`/`sandbox`, §3), **LSP 내비게이션**(`lsp_*`, §5), **웹 검색**(`websearch`), **프롬프트 캐싱**(`cache_control`, 기본 켜짐 + 자동 폴백). 테스트 예시가 있는 기능/마일스톤 스펙은 [`SPEC.md`](SPEC.md), 내부 구조는 [`ARCHITECTURE.md`](ARCHITECTURE.md)에 있다.

**진짜로 범위 밖(현재).** 웹 UI나 호스티드 원격 공유 없음 — magi는 터미널 클라이언트다(공유는 서버가 아니라 git 기반 experience 저장소로, §7). 자동 컨텍스트 *랭킹*은 임베딩이 아니라 의도적으로 어휘적/결정론적(BM25-lite, §4)이라 벡터-DB 의존이 없다. 이는 채워야 할 갭이 아니라 범위 선택이다.
